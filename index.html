FIG. 01 — 178,279 hosts cataloged · OPSEC-hardened

Indian Government Domain Attack-Surface Scanner & VAPT Pipeline

A fully automated, OPSEC-hardened toolkit that discovers, validates, and triages vulnerabilities across Indian government web infrastructure (.gov.in / .nic.in). Cataloged 178,279 hosts, deep-crawled 33,599 live targets, and surfaced 1,904 triaged findings — 120 Critical, 968 High — including exposed .git/ repos, .env files, live AWS key pairs, and private keys embedded in production JS bundles.

Every external request routes through a rotating 8-node proxy pool with DoH-over-proxy DNS, so the operator's real IP never touches a target; a periodic CI audit flags any bare network call. 24,468 LOC across 129 Python files.

GX (Genexis) Earth GPON Devices & ISP Infrastructure

Vulnerability Note

Security research across GX Earth GPON fiber-broadband devices and the associated ISP management infrastructure, coordinated through CERT-In. Three issues were assigned CVEs.

Hardcoded RSA private key in firmware

An identical RSA private key shipped across all devices, enabling HTTPS decryption and man-in-the-middle attacks.

CVSS 4.08.7
High
Critical · Remediated CVSS 3.1 · 9.8 · No CVE assigned · ~100,000+ devices in scope
ISP TR-069 Auto-Configuration Server (ACS)

The same research surfaced a fleet-wide mass-compromise path in Asianet Broadband's internet-facing TR-069 ACS — reported to CERT-In and since remediated. Combined, four weaknesses opened the entire customer fleet:

  • Unauthenticated registration — enrolment validated only by manufacturer OUI, so a spoofed device could register.
  • Universal credentials — identical connection-request credentials pushed to every router; one recovered set reached the whole fleet.
  • Unencrypted CWMP — TR-069 traffic ran over plain HTTP, exposing management credentials in transit.
  • Hardcoded credentials — firmware and ISP-provisioned credentials stored in device config (CWE-798 / CWE-312).
